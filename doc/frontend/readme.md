# Guide frontend

Ce dossier explique comment travailler sur l’interface utilisateur.

## Structure frontend

- views/ : fichiers HTML/PHP des pages
- views/layouts/ : templates généraux
- public/assets/css/ : styles
- public/assets/js/ : scripts JavaScript
- public/assets/images/ : images et icônes

## Comment comprendre une page

1. ouvrir la vue correspondante dans views/
2. repérer le layout utilisé
3. vérifier les classes CSS utilisées
4. modifier le contenu ou les styles si besoin

## Exemple de flux

- la route affiche une vue
- la vue utilise un layout
- le CSS donne la forme visuelle
- le JS ajoute des interactions si nécessaire

## Conseils pour les débutants

- commencez par les vues déjà présentes
- évitez de réécrire toute l’interface d’un coup
- testez chaque modification dans le navigateur
